Kotlin学习

Posted on 2019-12-04

包

目录与包的结构无需匹配，源代码可以在文件系统的任意位置。

包声明

包的声明应处于源文件顶部。
源文件所有内容，无论是类还是函数，都包含在声明的包内。
如果没有指明包，该源文件的内容属于无名字的默认包。

包导入

默认导入

有多个包会默认导入到每个 Kotlin 源文件中。

包 解释
kotlin.* Kotlin
kotlin.annotation.* 注解
kotlin.collections.* 集合
kotlin.comparisons.* 比较
kotlin.io.* 输入输入出
kotlin.ranges.* 区间
kotlin.sequences.* 序列
kotlin.text.* 文本
java.lang.* JVM平台额外导入–Java语言
kotlin.jvm.* JVM平台额外导入–JVM
kotlin.js.* JS平台额外导入–JS

手动导入

导入单独名称

导入作用域下所有内容

如果出现导入名称冲突，可以使用 as 关键字在本地重命名冲突项来消歧义

扩展

导入声明
顶层函数以及属性
在对象声明中声明的函数和属性
枚举常量

顶层声明的可见性
如果顶层声明是 private 的，它是声明它的文件所私有的

可见性修饰符

包

函数、属性和类、对象和接口可以在顶层声明，即直接在包内声明

可见性修饰符 解释
private 只会在声明它的文件内可见
protected 不适用于顶层声明
internal 会在相同模块内随处可见
public 默认可见性，此声明将随处可见

类和接口

类内部成员的声明

可见性修饰符 解释
private 只在这个类内部可见，外部类不能访问内部类的 private 成员
protected 只在这个类内部可见，如果覆盖一个 protected 成员并且没有显式指定其可见性，该成员还会是 protected 可见性
internal 能见到类声明的 本模块内 的任何客户端都可见其 internal 成员
public 能见到类声明的任何客户端都可见其 public 成员

构造函数

指定一个类的的主构造函数的可见性

可见性修饰符 解释
private 私有的
protected 受保护的
internal 一个 internal 类的构造函数只能在相同模块内可见
public 默认情况下，所有构造函数都是 public，这实际上等于类可见的地方它就可见

局部声明

局部变量、函数和类不能有可见性修饰符。

模块

可见性修饰符 internal 意味着该成员只在相同模块内可见。
更具体地说， 一个模块是编译在一起的一套 Kotlin 文件。

基本类型

数字

整数

类型 大小（比特数） 最小值 最大值
Byte 8 -128 127
Short 16 -32768 32767
Int 32 -2,147,483,648 (-2^31) 2,147,483,647 (2^31 - 1)
Long 64 -9,223,372,036,854,775,808 (-2^63) 9,223,372,036,854,775,807 (2^63 - 1)

未超出 Int 最大值的整型值初始化的变量都会推断为 Int 类型。
如果初始值超过了其最大值，那么推断为 Long 类型。
如需显式指定 Long 型值，请在该值后追加 L 后缀。

浮点数

类型 大小（比特数） 有效数字比特数 指数比特数 十进制位数
Float 32 24 8 6-7
Double 64 53 11 15-16

Float 反映了 IEEE 754 单精度，而 Double 提供了双精度。

对于以浮点数初始化的变量，编译器会推断为 Double 类型。 如需将一个值显式指定为 Float 类型，请添加 f 或 F 后缀。 如果这样的值包含多于 6～7 位十进制数，那么会将其舍入。

Kotlin 中的数字没有隐式拓宽转换。 例如，具有 Double 参数的函数只能对 Double 值调用，而不能对 Float、 Int 或者其他数字值调用。
